TIBCO

The Information Bus Company (TIBCO) is a software company that provides business integration software to integrate, manage and monitor enterprise applications and information delivery.

It contains a suite of products like TIBCO Enterprise Messaging Service (EMS), TIBCO Business Works(BW), TIBCO Administrator etc.

1. TIBCO EMS is an extension of Java Message service and is used for communication with other applications.

2. TIBCO BW is a package provided with in TIBCO suite of products where the integration logic is designed and developed.

3. TIBCO Administrator is used to deploy the application.

Business Works 5.x

1. It allows the automation of business processes .

2. It mediates interactions between different applications and databases .

3. Manage transactions and Web Services, Handle exceptions and report errors .

4. Provides graphical user interface to configure application services .

5. Provides plug-ins for application connectivity .

6. Provides interface for administrator to monitor and manage processes and application Resources

TIBCO

Installation of TIBCO suite of products (TRA, RV, EMS, BW, Admin) :

Below is the sequence of Installation of TIBCO suite of Products –

• TIBCO Rendezvous (RV) (v 8.5.0)

• TIBCO Runtime Agent (TRA) (v 5.11.0)

• TIBCO Enterprise Message Service (EMS) (v 8.5.1)

• TIBCO ActiveMatrix BusinessWorks (BW) (5.14.0)

• TIBCO Administrator - Enterprise Edition (Admin) (v 5.11.1)

• TIBCO Hawk (v 6.2.0)

Other products like adaptors (ADB, File, SAP R3) and Plugins (REST JSON, MQ, Salesforce) can also be installed based on project specific Requirements.


Modes of Installation :

• GUI Mode – Pictorial way of Installation.

• Console Mode – Interactive way of Installation.

• Silent Mode – Executed from Command Line, can be used for Scripted Installation.

Installation Prerequisites :

Always make sure to check the Product dependencies and compatibilities before installing the products.

Few thing to look for are – Platform or OS Type and version supported, Java version, Third Party Software support, Disk Space and Memory required and other TIBCO Product/Library dependencies.



Note –

To access databases from driver classes that are not provided by TIBCO Database Driver Supplements or to use the product specific drivers, the required driver library files has to be placed in the TPCL folder or the location of the drivers file has to be updated in the designer.tra (for testing locally during the development phase) and in bwengine.tra file (in deployment environment like DEV, UAT, PROD).




Schemas

A schema is a model for describing the structure of information.

It's a term borrowed from the database world to describe the structure of data in relational tables.

In the context of XML, a schema describes a model for a whole class of documents.

An XML schema defines the elements that an XML file may contain.

It provides a specific structure for XML data, which is important when sharing XML files between multiple systems.

Defining an XML schema ensures an XML Document will not contain unknown values, which may cause parsing errors.

The purpose of a schema is to allow machine validation of document structure.

Every specific, individual document which doesn't violate any of the constraints of the model is, by definition, valid according to that schema.
